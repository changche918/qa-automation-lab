Claude Code 的「rules」長什麼樣?

CLAUDE.md 實例 · 套用到你的爬蟲專案

如果套用到你之前做的 Bahamut / PTT 爬蟲專案,CLAUDE.md 大概會長這樣:

# 巴哈 / PTT 論壇爬蟲

## 專案概述
Python + Selenium 爬蟲,抓取巴哈姆特與 PTT 的高 GP 文章。
# GP = 論壇的「站內貨幣」,代表熱度

## 技術棧
- Python 3.12
- Selenium 4.x (Chrome headless)
- 套件管理用 uv,不用 pip

## 指令
- uv run python main.py — 執行爬蟲
- pytest tests/ -v — 跑測試
- ruff format . — 格式化

## 編碼慣例
- 繁體中文註解
- 函式名用 snake_case
- 新函式必須先寫 test 再實作
- GP 解析邏輯統一走 _parse_gp_value(),不要重複造輪子
- 爬到的資料存 dict,不要存格式化字串

## 資料結構
scraped item 必須長這樣:
{"gp": int, "title": str, "url": str}

## 安全規則
- 禁止 rm -rf 與 git push --force
- 爬蟲請求間隔 ≥ 2 秒,避免被 ban
- 不要硬編碼 cookie / session token

## 已知問題
- SSL handshake 偶爾 net_error -101,需要 retry 邏輯
- 巴哈舊版 HTML 結構不同,parse 前先判斷版本

這份檔案放在專案根目錄,每次 claude 啟動就自動讀入,相當於每次對話前自動幫你打了一段詳細的 system prompt。

常見疑問

FAQ
  • Q1. 所以「rules」不是 Claude Code 的正式術語?
    對。Anthropic 官方叫它 "memory file" 或 "instructions",不是 rules。「rules」是 Cursor 先用起來的說法,後來大家混著講。
    你聽到「Claude Code 的 rules」時,指的就是 CLAUDE.md。
  • Q2. CLAUDE.md 和 AGENTS.md 要擇一用嗎?
    不用。可以並存。建議策略:
    • 團隊共用的內容 (技術棧、指令、慣例) 放 AGENTS.md,其他 AI 工具也讀得到
    • Claude 專屬 的內容 (subagent 編排、Claude 特有功能) 放 CLAUDE.md
    • CLAUDE.md 開頭寫 See @AGENTS.md 引用即可
  • Q3. rules 和 Skills 有什麼差別?
    Rules (CLAUDE.md) 是「永遠套用的背景知識」 — 每次對話開頭就載入,不用觸發。
    Skills 是「按需觸發的工作流」 — 有名字、有描述,Claude 判斷需要時才載入。
    類比:rules 是釘在牆上的團隊公約,skills 是書架上的工具書。
  • Q4. rules 可以強制執行嗎?
    CLAUDE.md 本身不是強制 — 它是給 LLM 看的提示,Claude 通常會遵守但不保證。
    要硬性強制的規則 (例如「禁止跑 rm -rf」) 要用 Hooks,由程式邏輯攔截,不經過 AI。
  • Q5. 寫 rules 的常見錯誤?
    • 太長 — 200 行不如 50 行精準。超過 500 行 Claude 會開始忽略末端
    • 寫 linter 會做的事 — 「用兩格縮排」是 Prettier 的工作,別寫進去
    • 跨工具重複 — 不要 CLAUDE.md 和 .cursorrules 各寫一份,用 AGENTS.md 當單一來源
    • 用 /init 自動產生 — 產出通常又臭又長又不準,手寫的每行都有意義
