① CLAUDE.md · 專案的「長期記憶」

Project Memory File
是什麼

專案根目錄的 markdown 檔 ./CLAUDE.md

每次在該目錄啟動 Claude Code 時,內容會自動載入為 system prompt 的一部分。寫一次,所有後續對話都記得。

該寫什麼

專案慣例與規則

技術棧、資料夾結構、build/test 指令、命名規則、禁用行為、常用的 shell alias 等。精簡,不要塞文件全文。

範例

典型內容

# 專案: Bahamut 爬蟲
技術棧: Python 3.12, Selenium
建議用 uv 管理套件

## 測試
pytest tests/ -v

## 規則
- 禁止 rm -rf
- 新函式先寫 test
- 中文註解用繁中
層級

兩層記憶

全域 ~/.claude/CLAUDE.md — 個人偏好,每個專案都適用 (例如「用繁中回答」)。

專案 ./CLAUDE.md — 這個 repo 特有的規則,建議 commit 到 git 讓團隊共用。

② Skills / Slash Commands · 可重用的「工作流」

Unified in 2026 · 已統一為同一系統
2026 重要更新

Commands 與 Skills 合併

.claude/commands/deploy.md 和 .claude/skills/deploy/SKILL.md 現在都會產生同一個 /deploy 指令,運作方式相同。

建議改用 Skills 格式 — 支援資料夾、附屬檔案、YAML frontmatter 控制行為。

兩種觸發方式

手動 vs 自動

手動: 在終端打 /deploy,像指令一樣執行。

自動: Claude 讀到你需求描述,判斷某個 skill 相關就自動呼叫。用 frontmatter 的 disable-model-invocation 可關閉自動觸發。

內建 Skills

開箱即用

/debug — 系統化除錯流程
/simplify — 簡化複雜程式碼
/review — 自動 code review
/batch — 批次重複任務
/loop 5m /x — 每 5 分跑一次
/claude-api — Claude API 查詢

③ Subagents · 分身並行處理

Isolated AI Personas
核心價值

保護主線對話不被污染

Subagent 是獨立的 Claude 實例,有自己的 context window。主代理派任務給它,它獨自完成後只回傳結論,過程中的一堆讀檔、搜尋噪訊不會進入主對話。

典型用途

什麼情境該用

Explore: 掃描整個 codebase 找相關檔案 — 讀很多但只回傳地圖。

文件查詢: 抓 Dexie / 某 library 的文件,讀 10 頁只回摘要。

並行 review: 同時開多個 subagent 審查不同檔案。

Plan Mode

自動啟用 subagent

進入 plan mode (Shift+Tab 或 /plan) 時,Claude Code 預設會派 Explore-style subagent 去掃 repo,再把精華回傳給你規劃用。

非同步

Fire and forget

可以派出 subagent 後繼續做其他事,它在背景跑完會回來通知你。適合重型重構、大量測試生成等耗時任務。

④⑤⑥ 其他三個核心:MCP / Hooks / 內建指令

External Integration · Automation · Management
④ MCP Servers

連外部工具

Model Context Protocol — 讓 Claude 安全地使用外部服務。官方或社群已有 GitHub、Postgres、Playwright、Slack 等 server。

vs Skills 差別: MCP 是「連線能力」,Skills 是「使用知識」。兩者互補。

⑤ Hooks

事件自動化

在特定事件觸發腳本,不經過 AI 判斷 — 確定性執行。

例如:每次編輯 .py 後自動 ruff format、commit 前跑 pytest、偵測到 rm -rf 立刻阻擋。

⑥ 核心 Slash Commands

對話管理工具

/compact — context 快滿時壓縮歷史。可加 focus:/compact 保留爬蟲邏輯

/context — 視覺化看 context 用了多少。

/diff — 查看 Claude 所有修改。

組合起來

典型架構

CLAUDE.md 定規範 → Skills 提供工作流 → Subagents 處理重活 → MCP 接外部 → Hooks 把關品質 → Slash commands 管 context。

不用全部開。先一個 CLAUDE.md + 一個 skill 就夠用,再慢慢疊。

Claude Code 建議入門流程

  1. 安裝: npm i -g @anthropic-ai/claude-code,然後在專案資料夾打 claude 啟動。
  2. 寫第一個 CLAUDE.md: 在專案根目錄建立,列出技術棧、build/test 指令、3-5 條最重要的規則。保持精簡。
  3. 先從小任務試水溫: 寫一個既有函式的測試、修已知的 bug。觀察 Claude Code 怎麼讀你的 codebase。
  4. 加一個 MCP server: 挑你每天用的工具 (GitHub / 資料庫 / 搜尋引擎) 接起來。
  5. 寫第一個 Skill: 從你常重複的工作流開始 — 例如 code review checklist、deploy 流程、Bahamut 爬蟲模板。
  6. 加 Hooks 自動化善後: 最常見的是編輯後自動 lint、commit 前自動測試。
  7. 熟了之後玩 Subagents: 跨多檔重構、大規模測試生成、並行 review — 讓 subagent 去做,主線保持乾淨。
  8. Context 管理: 對話長了就 /compact;快滿時看 /context 確認狀況;不想失去快取就不要隨便 /clear。
